# Duffing Unforced Double-Well Smoke Files

## Task Summary

This task adds the smoke-stage engineering path and a formal entry point for
the `v1_core` `duffing_unforced_double_well` system. The system is the
unforced damped double-well Duffing oscillator with full-state clean
observation:

```text
q_dot = v
v_dot = -delta * v - alpha * q - beta * q^3
z = x = [q, v]
```

Both the smoke and current formal runs use the same smoke-scale parameters:
`delta = 0.2`, `alpha = -1.0`, `beta = 1.0`, eight manual initial
conditions, `dt = 0.02`, and 800 time steps.

## Run Entry Points And Scripts

- `experiments/smoke_tests/run_duffing_unforced_double_well_smoke.jl`
  loads the smoke configs, generates raw trajectories, applies full-state
  observation, builds Split-I and window summaries, runs diagnostics, saves
  outputs, and prints a completion summary.
- `experiments/data_generation/generate_duffing_unforced_double_well_formal_dataset.jl`
  is the formal generation entry point. It intentionally reuses the smoke
  system, split, window, and task configs, then overrides only the benchmark
  id, difficulty label, and output paths to write into `formal/` locations.

Run command:

```powershell
julia --project=. experiments/smoke_tests/run_duffing_unforced_double_well_smoke.jl
```

Formal command:

```powershell
julia --project=. experiments/data_generation/generate_duffing_unforced_double_well_formal_dataset.jl
```

## Core Source, Configs, And Docs Changed

- `src/dynamics/duffing.jl`: Duffing spec, validation, vector field, energy,
  equilibria, Jacobian, fixed-step RK4 propagation, and metadata.
- `src/diagnostics/duffing_diagnostics.jl`: finite checks, full-state
  observation error, energy monotonicity checks, final well classification,
  state ranges, window/split count enrichment, and smoke pass policy.
- `src/generators/duffing_dataset_generator.jl`: manual smoke initial
  conditions, raw and observed trajectory construction, split/window summary,
  JLD2 and JSON output, difficulty-labeled plots/tables/logs, release index,
  and log writing.
- `experiments/data_generation/generate_duffing_unforced_double_well_formal_dataset.jl`:
  formal dataset entry point using smoke-scale parameters.
- `configs/systems/v1_core/duffing_unforced_double_well_smoke.json`: smoke
  system and integration settings.
- `configs/observations/duffing_full_state_clean.json`: full-state clean
  observation.
- `configs/splits/v1_core/duffing_smoke_split_i.json`: trajectory-level
  initial-condition split.
- `configs/windows/v1_core/duffing_smoke_windows.json`: one-step, rollout,
  and statistics windows.
- `configs/tasks/v1_core/duffing_smoke_tasks.json`: smoke task declarations.
- `configs/benchmarks/v1_core/duffing_smoke_benchmark.json`: smoke benchmark
  and output policy.

## Script-To-Script Data Flow

The smoke script follows this order:

1. Read benchmark, system, observation, split, window, and task configs.
2. Convert the system config into `DuffingUnforcedDoubleWellSpec`.
3. Generate eight raw state trajectories as `RawTrajectory` objects.
4. Apply identity full-state observation to produce `ObservedTrajectory`
   objects.
5. Build trajectory-level Split-I, then derive one-step, rollout, and
   statistics window summaries inside each split.
6. Compute Duffing diagnostics, including energy monotonicity and final well
   membership.
7. Save data, manifest, table, plots, and log files under the configured
   `smoke/` or `formal/` output paths.
