# Duffing Unforced Double-Well Smoke Files

## Task Summary

This task adds the smoke-stage engineering path for the `v1_core`
`duffing_unforced_double_well` system. The system is the unforced damped
double-well Duffing oscillator with full-state clean observation:

```text
q_dot = v
v_dot = -delta * v - alpha * q - beta * q^3
z = x = [q, v]
```

The smoke run uses the default parameters `delta = 0.2`, `alpha = -1.0`,
`beta = 1.0`, eight manual initial conditions, `dt = 0.02`, and 800 time
steps.

## Run Entry Points And Scripts

- `experiments/smoke_tests/run_duffing_unforced_double_well_smoke.jl`
  loads the smoke configs, generates raw trajectories, applies full-state
  observation, builds Split-I and window summaries, runs diagnostics, saves
  outputs, and prints a completion summary.

Run command:

```powershell
julia --project=. experiments/smoke_tests/run_duffing_unforced_double_well_smoke.jl
```

- `configs/observations/duffing_full_state_clean.json`: full-state clean

- `configs/windows/v1_core/duffing_smoke_windows.json`: one-step, rollout,

- `configs/tasks/v1_core/duffing_smoke_tasks.json`: smoke task declarations.

## Script-To-Script Data Flow

The smoke script follows this order:

1. Read benchmark, system, observation, split, window, and task configs.
2. Convert the system config into `DuffingUnforcedDoubleWellSpec`.
3. Generate eight raw state trajectories as `RawTrajectory` objects.
4. Apply identity full-state observation to produce `ObservedTrajectory`
   objects.
5. Build trajectory-level Split-I, then derive one-step, rollout, and
   statistics window summaries inside each split.
6. Compute Duffing diagnostics, including energy monotonicity and final well
   membership.
7. Save data, manifest, table, plots, and log files.
